# データ管理3層 — 実装タスク整理

## 確定したサイドバー構成

```
帳簿:       仕訳帳 / 総勘定元帳 / 試算表         （変更なし）
決算書類:   損益計算書 / 貸借対照表 / 消費税集計 / 固定資産台帳  （変更なし）
確定申告:   青色申告決算書                        （変更なし）
出力:       帳簿出力 /reports    / アーカイブ /archive  ← 追加
請求書:     請求書一覧 / 取引先管理               （変更なし）
設定:       勘定科目 /accounts   / 設定 /settings  ← 変更
データ管理: データ管理 /data                       ← 新セクション
```

### ルート変更まとめ

| ルート      | 状態     | 内容                                                       |
| ----------- | -------- | ---------------------------------------------------------- |
| `/settings` | **新規** | 事業者情報 + 証憑保存先 + ストレージ容量                   |
| `/archive`  | **新規** | 検索機能付アーカイブ生成                                   |
| `/data`     | **改修** | バックアップ・リストア + インポート・エクスポート（3層UI） |
| `/accounts` | 変更なし | 勘定科目管理                                               |
| `/reports`  | 変更なし | 帳簿出力                                                   |

### `/data` から `/settings` に移動するコンポーネント

| コンポーネント        | 現在    | 移動先      |
| --------------------- | ------- | ----------- |
| `BusinessInfoCard`    | `/data` | `/settings` |
| `StorageSettingsCard` | `/data` | `/settings` |
| `CapacityCard`        | `/data` | `/settings` |

### `/data` に残るコンポーネント（3層に再構成）

| コンポーネント       | レイヤー                 |
| -------------------- | ------------------------ |
| `BackupCard`（新規） | バックアップ・リストア   |
| `ExportCard`（改修） | インポート・エクスポート |
| `ImportCard`（改修） | インポート・エクスポート |

## 技術メモ

- **BackupCard**: `zip-export.ts` の `exportToZip()` + `zip-import.ts` を流用
- **ImportCard 改修**: ZIPファイル選択時は「バックアップ・リストアからリストアしてください」と誘導
- **`/settings` の状態共有**: storageMode/directoryHandle は store（`$state` rune）で管理し、`/data` でも参照
- **アーカイブ `index.html`**: Vanilla JS（外部依存なし）。`data.json` は `<script>` タグで埋め込み
- **既存コード活用**: `zip-export.ts` の証憑収集ロジック、`ExportCard.svelte` の進捗UIパターンを再利用
