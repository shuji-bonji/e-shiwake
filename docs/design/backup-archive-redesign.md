# v0.4.0 バックアップ・アーカイブ改善仕様

## 背景

v0.3.1 までのバックアップは「年度のバックアップ」という体裁だが、実際にはグローバルデータ（勘定科目・取引先・固定資産・設定・事業者情報）を全件含むフルスナップショットになっていた。この曖昧さにより以下の問題がある:

- 複数年度を順番にリストアすると、グローバルデータが後勝ちで上書きされる
- 事業者情報は年度ごとに変わりうる（免税→課税、事業所移転、屋号変更等）
- アーカイブ後に年度削除しても請求書やローカル証憑ファイルが残る
- マージ/上書きモードでグローバルデータの競合が定義しきれない

## 設計方針

各機能の役割を明確に分離する:

| 機能             | スコープ | 目的                                   | リストアモード |
| ---------------- | -------- | -------------------------------------- | -------------- |
| **バックアップ** | 全データ | スナップショット（端末移行・事故対策） | 上書きのみ     |
| **アーカイブ**   | 年度別   | 年度パッケージ保存 + 年度データ復活    | マージのみ     |
| **エクスポート** | 年度別   | CSV/JSON外部出力（参照用）             | —              |

## 1. バックアップ（フルスナップショット）

### 1.1 バックアップ作成

- 年度選択を廃止。全年度の全データを一括バックアップ
- 新しい型 `BackupData` を使用（`ExportData` とは別）
- ファイル名: `e-shiwake_backup_YYYY-MM-DD.zip`
- UIは単一の「バックアップ作成」ボタン + 全体サマリ表示

#### BackupData に含まれるデータ

| データ           | スコープ |
| ---------------- | -------- |
| 仕訳             | 全年度   |
| 証憑PDF          | 全年度   |
| 請求書           | 全年度   |
| 固定資産         | 全件     |
| 勘定科目         | 全件     |
| 取引先           | 全件     |
| 設定・事業者情報 | 全件     |

### 1.2 リストア

- マージモードを廃止 → **上書きのみ**
- 「全データを上書きします」の確認ダイアログ（強い警告）
- 証憑保存先は1回選択 → 全年度の `storageModeByYear` に展開
- グローバルデータ全て上書き復元
  - 除外: `storageMode`, `storageModeByYear`, `lastExportedAt`（リストア先環境に依存するため）

### 1.3 BackupData 型（新規）

```typescript
interface BackupData {
	type: 'backup'; // ZIP判別用
	version: '3.0.0';
	exportedAt: string;
	journals: JournalEntry[]; // 全年度
	accounts: Account[]; // 全件
	vendors: Vendor[]; // 全件
	fixedAssets: FixedAsset[]; // 全件
	invoices: Invoice[]; // 全年度
	allSettings: Partial<SettingsValueMap>;
}
```

## 2. アーカイブ

### 2.1 アーカイブ作成

変更なし（現行維持）。年度別にZIPを生成。

### 2.2 年度データ削除の強化

現行の `deleteYearData` を拡張:

| データ                    | 現行    | 変更後                                           |
| ------------------------- | ------- | ------------------------------------------------ |
| 仕訳                      | ✅ 削除 | ✅ 削除                                          |
| 添付ファイル（IndexedDB） | ✅ 削除 | ✅ 削除                                          |
| 証憑ファイル（ローカル）  | ❌ 残る | ✅ ハンドルがあれば削除（なければスキップ+警告） |
| 請求書                    | ❌ 残る | ✅ 該当年度分を削除                              |
| 固定資産                  | ❌ 残る | ❌ 残す（複数年度にまたがるため）                |
| 取引先                    | ❌ 残る | ❌ 残す（グローバルデータ）                      |
| 勘定科目                  | ❌ 残る | ❌ 残す（グローバルデータ）                      |
| 設定                      | ❌ 残る | ❌ 残す（グローバルデータ）                      |

### 2.3 アーカイブリストア（新機能）

- アーカイブZIPを選択 → 仕訳＋証憑のみマージ復元
- グローバルデータは一切触らない
- 証憑保存先は年度単位で選択
- ZIP内のレポート類（HTML/CSV）は無視
- ヘルプに「グローバルデータ（勘定科目・取引先・設定等）は復元されません」と明記

## 3. エクスポート

変更なし。年度別のCSV/JSON出力として維持。

- CSV: 仕訳のみ
- JSON: `ExportData` v3（年度スコープ、証憑なし）
- 年度削除機能はエクスポートカードに残す

## 4. ZIP判別と後方互換

### 4.1 判別ロジック

```
data.json に "type": "backup" がある
  → BackupData → フルリストア

data.json に "fiscalYear" がある
  → ExportData (v1/v2/v3) → アーカイブリストア
```

### 4.2 後方互換マトリクス

| ZIPの出自                     | フォーマット       | リストア方法       | 動作                     |
| ----------------------------- | ------------------ | ------------------ | ------------------------ |
| 新バックアップ（v0.4.0〜）    | `BackupData` v3    | フルリストア       | 全データ上書き           |
| 旧バックアップ v2（v0.3.x）   | `ExportData` v2    | アーカイブリストア | 年度の仕訳＋証憑をマージ |
| 旧バックアップ v1（〜v0.2.x） | `ExportData` v1    | アーカイブリストア | 年度の仕訳＋証憑をマージ |
| アーカイブ                    | `ExportData` v2/v3 | アーカイブリストア | 年度の仕訳＋証憑をマージ |

### 4.3 旧バックアップの扱い

旧バックアップ（v1/v2）はアーカイブリストアで読み込まれる。
ZIP内のグローバルデータ（勘定科目・取引先・設定）は無視され、仕訳＋証憑のみ復元される。
グローバルデータも含めて完全復元したい場合は、新フォーマットでバックアップを取り直す必要がある。

## 5. ユーザーフロー

### 通常運用

```
日々の作業 → 定期的にフルバックアップ（スナップショット保存）
```

### 確定申告後

```
確定申告完了 → アーカイブ作成 → 年度データ削除（容量節約）
```

### 過年度データが必要になった場合

```
仕訳の確認・修正が必要
  → アーカイブからリストア（年度データだけ復活、グローバル設定はそのまま）

全環境を過去の状態に戻したい
  → フルバックアップからリストア（全データ上書き）
```

## 6. UI変更まとめ

### データ管理ページ（`/data`）

**BackupCard:**

- 年度一覧を廃止 → 単一の「バックアップ作成」ボタン + 全体サマリ（全年度の仕訳数・証憑数・請求書数等）
- リストアのマージ/上書き選択を廃止 → 上書きのみ（確認ダイアログ強化）
- ZIP判別: BackupData → そのままフルリストア / ExportData → 「アーカイブページでリストアしてください」と誘導

**ExportCard:**

- 変更なし

### アーカイブページ（`/archive`）

- 「アーカイブからリストア」セクションを追加
  - ZIPファイル選択 → プレビュー（年度・仕訳数・証憑数）→ 証憑保存先選択 → マージ復元
  - 旧バックアップ（v1/v2）ZIPも受け付ける
