# デグレリスクが高い箇所

## 1. DBマイグレーション（最重要）

Version 7でvendorsテーブルにcreatedAt/updatedAtを追加するマイグレーションがあります。

### 確認項目：

- 既存の取引先データが正しく表示されるか
- 仕訳帳で取引先名が正しく表示されるか
- 取引先のオートコンプリートが動作するか

## 2. 仕訳機能（既存のコア機能）

isVendorInUseByJournal関数が既存の仕訳データを参照します。

### 確認項目：

- 仕訳の新規作成・編集・削除
- 証憑の添付・削除
- 仕訳のコピー機能
- 年度フィルタリング

## 3. 印刷機能（layout.css変更）

@media printに請求書用CSSを追加しました。

### 確認項目：

- 仕訳帳の印刷
- 総勘定元帳の印刷
- 試算表・損益計算書・貸借対照表の印刷
- 青色申告決算書の印刷

## 4. 設定・データ管理ページ

事業者情報カードを追加しました。

### 確認項目：

- エクスポート（JSON/CSV/ZIP）が正常に動作するか
- インポートが正常に動作するか
- ストレージ使用量の表示

## 推奨テスト手順

1. アプリを開く（DBマイグレーション実行）
2. 仕訳帳で既存データが表示されるか確認
3. 新規仕訳を作成（取引先オートコンプリート確認）
4. 総勘定元帳・試算表を表示
5. 各帳簿の印刷プレビューを確認
6. データ管理ページでJSONエクスポート
7. 請求書機能を試す
   特にDBマイグレーションと印刷機能は、実際に動かして確認することをお勧めします。
